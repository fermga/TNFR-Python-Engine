**Status**: Production-ready ✅

## Overview

This guide documents the **complete self-optimization integration** across TNFR modules, providing a unified workflow for:

1. **Paley Factorization** (original integration)
2. **Pattern Discovery** (mathematical pattern engine)
3. **Fractal Partitioning** (coherence-based network partitioning)
4. **Batch Certificate Processing** (analysis workflows)

All integrations use the **centralized self-optimization support** module (`tnfr_factorization.self_opt_support`) and expose clean APIs via the **TNFR SDK** (`tnfr.sdk`).

## Usage Examples

### 1. Pattern Discovery Optimization

```python
from tnfr.engines.pattern_discovery import TNFREmergentPatternEngine
from tnfr.sdk import run_pattern_discovery_optimization
import networkx as nx

# Create and analyze network
G = nx.karate_club_graph()
for node in G.nodes():
    G.nodes[node]["vf"] = 1.0
    G.nodes[node]["phase"] = 0.0
    G.nodes[node]["EPI"] = 1.0

# Discover patterns
engine = TNFREmergentPatternEngine()
discovery_result = engine.discover_all_patterns(G)

# Export manifest
manifest_info = engine.export_pattern_manifest(
    G=G,
    discovery_result=discovery_result,
    output_dir="results/patterns",
    partition_id="experiment_42",
)

# Run self-optimization
opt_result = run_pattern_discovery_optimization(
    manifest_path=manifest_info["manifest_absolute"],
    manifest_summary_path=manifest_info["summary_absolute"],
    base_name="pattern_opt_42",
)

print(f"Optimization result: {opt_result}")
```

### 2. Fractal Partition Optimization

```python
from tnfr.parallel import FractalPartitioner
from tnfr.sdk import run_fractal_partition_optimization
import networkx as nx

# Create network
G = nx.karate_club_graph()
for node in G.nodes():
    G.nodes[node]["vf"] = 1.0
    G.nodes[node]["phase"] = 0.0

# Partition with manifest
partitioner = FractalPartitioner(max_partition_size=10)
result = partitioner.partition_with_manifest(
    graph=G,
    output_dir="results/partitions",
    partition_id="network_42",
)

# Run self-optimization
opt_result = run_fractal_partition_optimization(
    manifest_path=result["manifest_absolute"],
    manifest_summary_path=result["summary_absolute"],
    base_name="partition_opt_42",
)

print(f"Partitions: {len(result['partitions'])}")
print(f"Optimization result: {opt_result}")
```

### 3. Batch Certificate Optimization

```python
from factorization_lab.analysis.analyze_patterns import generate_optimization_manifest
from tnfr.sdk import run_batch_certificate_optimization

# Generate optimization manifest from certificate analysis
manifest_info = generate_optimization_manifest(
    certificate_manifest_path="results/certificates/certificate_manifest.json",
    output_dir="results/batch_optimization",
    batch_id="batch_42",
    certificate_filter={"coherence_ratio_max": {"min": 0.7}},
)

# Run self-optimization on batch
opt_result = run_batch_certificate_optimization(
    manifest_path=manifest_info["manifest_absolute"],
    manifest_summary_path=manifest_info["summary_absolute"],
    base_name="batch_opt_42",
)

print(f"Optimization result: {opt_result}")
```

## CLI Pipeline Workflow

End-to-end workflow that takes partition manifests, produces dry-run recommendations,
validates them, and feeds qualified operator strategies into downstream orchestration.

### Generate Dry-Run Recommendations

```powershell
pwsh> python scripts/run_self_optimization.py `
    --manifest manifests/paley/_manifest.json `
    --output-dir results/self_optimization/paley_v3 `
    --operation-type paley_partition `
    --max-workers 4 `
    --capture-snapshots `
    --summary results/self_optimization/paley_v3_summary.json
```

Key flags:
- `--manifest` (required) — path to `_manifest.json` describing partitions.
- `--output-dir` — storage for per-partition payloads + telemetry.
- `--operation-type` — label recorded in each payload.
- `--max-workers` — parallelism for partition processing.
- `--capture-snapshots` — forces telemetry capture even when not in dry-run mode.
- `--summary` — writes aggregated summary JSON for CI artifacts.

### Validate Recommendations

```powershell
pwsh> python scripts/run_self_opt_validation.py `
    --payload-root results/self_optimization/paley_v3 `
    --report results/self_optimization_validation.json `
    --fail-on-regression
```

Arguments:
- `--payload-root` — root directory containing seed folders.
- `--report` — summary output (defaults to `results/self_optimization_validation.json`).
- `--fail-on-regression` — exits with status 1 if any pytest subset fails.

### Directory Layout

```
results/
  self_optimization/
    seed_alpha/
      paley_partition_0001.json
      paley_partition_0001.json.sha256
    seed_beta/
      ...
    paley_v3_summary.json
  self_optimization_validation.json
```

- **Seed folders** group payloads produced in a single runner invocation.
- **`.sha256`** files mirror payloads for integrity verification.
- **Summary JSON** aggregates per-run statistics.
- **Validation report** lists discovered payloads, mapped tests, and exit codes.

### JSON Schemas

**Partition payload** (`results/self_optimization/**/paley_partition_XXXX.json`):

```json
{
  "metadata": {
    "operation_type": "paley_partition",
    "partition_id": "0001",
    "manifest": "manifests/paley/_manifest.json",
    "seed": 41,
    "sequence": ["AL", "UM", "IL", "SHA"]
  },
  "engine": {
    "dry_run": true,
    "operator_strategy_plan": {
      "per_partition": {},
      "stabilizers": ["IL", "THOL"],
      "resonance_budget": 0.18
    },
    "telemetry_snapshots": [
      {
        "partition_id": "0001",
        "delta_phi_s": 0.42,
        "delta_c": 0.07,
        "fields": {"phi_s": 0.63, "grad_phi": 0.12, "k_phi": 1.46, "xi_c": 0.88}
      }
    ]
  },
  "telemetry": {
    "delta_phi_s": 0.42,
    "delta_c": 0.07,
    "coherence": 0.82,
    "sense_index": 0.76
  }
}
```

**Validation summary** (`results/self_optimization_validation.json`):

```json
{
  "payload_root": "results/self_optimization/paley_v3",
  "total_recommendations": 12,
  "status_counts": {"validated": 8, "regressed": 2, "pending": 2},
  "results": [
    {
      "path": "results/self_optimization/.../paley_partition_0001.json",
      "operation_type": "paley_partition",
      "status": "validated",
      "exit_code": 0
    }
  ]
}
```

Status meanings: `validated` (pytest passed), `regressed` (pytest failed), `pending` (no mapped tests).

### Promotion Guidance

1. **Confirm validation status** — only promote partitions marked `validated`.
2. **Telemetry guardrails** — ΔΦ_s < 0.6 and ΔC(t) ≥ +0.05 preferred. Reject payloads violating thresholds from `docs/STRUCTURAL_FIELDS_TETRAD.md`.
3. **Embed plan** — copy `operator_strategy_plan` into the target manifest or orchestration layer.
4. **Record provenance** — append manifest ID, git SHA, and validation report path to the promotion log.

### Review Checklist

| Step | Purpose | Command / Artifact |
|------|---------|--------------------|
| Telemetry sanity | Confirm Φ_s, \|∇φ\|, K_φ, ξ_C within bounds | Inspect `telemetry_snapshots` |
| ΔΦ_s / ΔC(t) trend | Coherence gains match potential drift | Plot via dashboard |
| Operator coverage | Sequences comply with U1-U6 | `tnfr.validation.grammar.collect_grammar_errors` |
| Validation sweep | Targeted pytest suites cover payloads | `python scripts/run_self_opt_validation.py --fail-on-regression` |
| Regression nets | Smoke targets green | `./make.cmd smoke-tests` |

### Dashboard Guidance (ΔΦ_s vs ΔC(t))

Extend `HealthVisualizationDashboard` (see `STRUCTURAL_HEALTH.md`) with a scatter overlay plotting ΔΦ_s (x) vs ΔC(t) (y). Color points by validation status; add threshold lines at ΔΦ_s = 1.0 (vertical) and ΔC(t) = 0 (horizontal). Store rendered dashboards under `results/reports/self_opt/`.

## Implementation Checklist

When adding self-optimization to a new module:

- [ ] **Create manifest export method** that generates JSON with:
  - `operation_type` field (unique identifier)
  - `partition_id` or equivalent identifier
  - `telemetry` dict with coherence, sense_index
  - `network_metadata` with node/edge counts
  - Module-specific data fields

- [ ] **Write summary file** alongside manifest with key metrics

- [ ] **Add SDK wrapper function** in `src/tnfr/sdk/self_opt.py`:
  ```python
  def run_<module>_optimization(...):

          ...,
          operation_type="<module>_operation"
      )
  ```

- [ ] **Expose in SDK** via `src/tnfr/sdk/__init__.py`:
  - Add to `__all__`
  - Add lazy import in `__getattr__`

- [ ] **Create unit tests** in appropriate `tests/` subdirectory:
  - Test manifest file creation
  - Test manifest structure validation
  - Test SDK wrapper integration
  - Test telemetry computation

- [ ] **Update documentation** in this file with:
  - Manifest format specification

## Troubleshooting

### Issue: "Self-optimization helpers not available"

**Cause**: `tnfr_factorization` package not installed or not in Python path.

**Solution**:
```bash
# Ensure factorization-lab is in PYTHONPATH
export PYTHONPATH="${PYTHONPATH}:$(pwd)/factorization-lab"

# Or install in development mode
pip install -e factorization-lab/
```

### Issue: "Manifest validation failed"

**Cause**: Missing required fields in manifest JSON.

**Solution**: Ensure manifest includes all common fields:
- `operation_type`
- `partition_id` (or equivalent)
- `telemetry` dict
- `network_metadata` dict

### Issue: "CLI runners not found"

**Cause**: Scripts not available or incorrect import paths.

**Solution**: Verify scripts exist:
```bash
ls scripts/run_self_optimization.py
ls scripts/run_self_opt_validation.py
```

    "python", "scripts/run_self_optimization.py",
