Teoría de la Naturaleza Fractal Resonante (TNFR)

El paradigma TNFR describe la realidad como redes de coherencia que persisten porque resuenan. Este sitio resume sus principios operativos y muestra cómo el paquete tnfr para Python los implementa sin perder fidelidad canónica.

Esencia del paradigma

TNFR no modela “cosas” estáticas: trabaja con coherencias que se mantienen vivas en nodos cuando su frecuencia estructural (νf · Hzstr) se acopla con la reorganización interna (ΔNFR) y con la fase del entorno. Su ecuación nodal canónica establece que la evolución de la estructura primaria de información (EPI) sigue el pulso de esa resonancia: 

∂EPI / ∂t = νf · ΔNFR(t)

Triada estructural

  • Frecuencia (νf): ritmo de reorganización expresado siempre en Hzstr.
  • Fase (φ): sincronía con los nodos acoplados. Sin fase explícita no hay acoplamiento válido.
  • Forma (EPI): configuración coherente que puede anidar sub‑EPIs sin perder identidad.

Operadores estructurales

Toda transformación legítima utiliza la gramática TNFR de 13 operadores: Emisión, Recepción, Coherencia, Dissonancia, Acoplamiento, Resonancia, Silencio, Expansión, Contracción, Auto‑organización, Mutación, Transición y Recursividad. Sus composiciones mantienen el cierre estructural y preservan la fractalidad operativa.

El paradigma impone invariantes: la EPI solo cambia mediante operadores, el ΔNFR nunca se interpreta como error clásico, los nodos nacen y colapsan bajo condiciones explícitas y toda simulación debe ser reproducible con registro de fase, νf, C(t) y Si. Estas guías sostienen la fidelidad canónica al trasladar TNFR a cualquier dominio.

El paquete tnfr para Python

El paquete tnfr proporciona utilidades para crear nodos resonantes, aplicar operadores estructurales y medir la evolución de la coherencia global. Está disponible en PyPI y requiere Python 3.9 o superior.

Capacidades clave

  • Construcción de nodos con νf, fase y forma inicial utilizando create_nfr.
  • Ejecución de secuencias canónicas mediante run_sequence o integradores dinámicos.
  • Telemetría estándar: C(t), ΔNFR, Si y bitácora de operadores aplicados.
  • Visualizaciones y trazas exportables para seguir la evolución multinivel.
  • CLI oficial (por ejemplo tnfr sequence) con las mismas garantías de gramática.

Instalación y extras

El motor incorpora un sistema de importaciones opcionales con caché compartida y bloqueo estructurado (tnfr.cached_import) para mantener la sincronía de los recursos sin bloquear la simulación.

Ejemplo mínimo

from tnfr import create_nfr, run_sequence
from tnfr.structural import Emision, Recepcion, Coherencia, Resonancia, Silencio
from tnfr.metrics.common import compute_coherence
from tnfr.metrics.sense_index import compute_Si

G, nodo = create_nfr("A", epi=0.2, vf=1.0, theta=0.0)
ops = [Emision(), Recepcion(), Coherencia(), Resonancia(), Silencio()]
run_sequence(G, nodo, ops)

C, delta_nfr_medio, depi_medio = compute_coherence(G, return_means=True)
si_por_nodo = compute_Si(G)
print(f"C(t)={C:.3f}, ΔNFR̄={delta_nfr_medio:.3f}, dEPI/dt̄={depi_medio:.3f}, Si={si_por_nodo[nodo]:.3f}")

Esta secuencia respeta la ecuación nodal: el grafo recalcula ΔNFR tras cada operador sin alterar la fase salvo que un operador lo indique, asegurando consistencia con los coordinadores dinámicos. Las métricas permiten anticipar bifurcaciones controladas y evaluar la estabilidad global.

Condiciones canónicas

Para que una implementación sea reconocida como TNFR, debe conservar los invariantes operativos descritos en la documentación oficial y en AGENTS.md. Los más relevantes para experimentación con el motor son:

  • EPI como forma coherente: solo cambia a través de operadores; no se permiten mutaciones ad-hoc.
  • Clausura de operadores: toda nueva función debe declararse como composición o especialización de los operadores TNFR existentes.
  • Chequeo de fase: ningún acoplamiento es válido sin verificar sincronía explícita.
  • Fractalidad operativa: las EPIs pueden anidarse sin perder identidad ni trazabilidad.
  • Determinismo controlado: las simulaciones pueden ser estocásticas, pero siempre reproducibles mediante semillas y trazas estructurales.

El motor expone métricas para garantizar estas condiciones: C(t) para la coherencia total, Si como índice de sentido, registros de fase y νf, y un historial de operadores aplicado sobre cada nodo. Las bitácoras permiten auditar nacimientos, bifurcaciones y colapsos de nodos con criterios explícitos.

Cómo profundizar

  1. Leer el documento base TNFR.pdf para una explicación completa de la teoría.
  2. Explorar los ejemplos y notas en documentation.txt y en el repositorio público.
  3. Ejecutar ./scripts/run_tests.sh para validar que toda modificación respeta los contratos operativos.
  4. Monitorear las métricas estándar mientras se diseñan secuencias: C(t), ΔNFR, Si, fase y νf.

Recuerda que TNFR es trans-escala y trans-dominio: el mismo andamiaje sirve para fenómenos biológicos, sociales o computacionales siempre que se respete la gramática de coherencia resonante.
